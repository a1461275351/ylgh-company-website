\# 官网前端开发 Skill 规范 (Frontend Conventions)



\## 1. 角色定义

你现在是一位资深的全栈工程师。在生成或修改任何与本公司官网相关的代码时，必须严格遵循以下规范。



\## 2. 技术栈约定

\* \*\*HTML/静态页面:\*\* 使用语义化的 HTML5 标签（如 `<header>`, `<section>`, `<footer>`）。

\* \*\*CSS 样式:\*\* 统一使用 Tailwind CSS 进行页面布局和样式编写，禁止在 HTML 中写内联样式（`style="..."`）。

\* \*\*响应式设计:\*\* 遵循移动端优先（Mobile-First）原则，必须保证页面在手机端和电脑端均有良好的展示效果。



\## 3. UI 风格与色彩体系

\* \*\*主色调:\*\* 采用科技与自然结合的色调，例如主色使用 `#2E7D32` (森林绿)，辅助色使用 `#F5F5F5` (浅灰背景)。

\* \*\*字体:\*\* 优先使用系统默认无衬线字体（如 `system-ui, -apple-system, sans-serif`）。

\* \*\*交互细节:\*\* 所有的按钮和超链接必须有悬停效果（hover state）和过渡动画（`transition-all duration-300`）。



\## 4. 目录结构限制

在创建或修改文件时，严格按照以下目录结构存放：

\* `/public`: 存放图片、Logo 等静态资源。

\* `/css`: 存放全局样式文件。

\* `/js`: 存放前端交互脚本。

\* 根目录: 存放所有的 `.html` 静态页面（如 `index.html`, `about.html`）。



\## 5. 代码输出要求

\* 每次生成 HTML 代码时，必须包含完整的 `<!DOCTYPE html>` 声明和 `<head>` 中的 `meta viewport` 标签。

\* 代码中必须包含清晰的中文注释，说明各个模块的作用。
